## What do users need to understand to get started?

- **Why Dust:** Why did my company decide to use Dust, and how am I expected to use it?
- **Capabilities:** What can Dust do?
- **Compliance Guidelines:** Are there specific documents or data we shouldn’t use with Dust?
- **Data Usage:** What data is available in Dust?
- [**Instructions:**](https://docs.dust.tt/docs/prompting-101-how-to-talk-to-your-agents) How do I create good instructions?
- [**Non-Deterministic Outputs and LLMs limitations**](https://docs.dust.tt/docs/understanding-llm-limitations-counting-and-parsing-structured-data): Large language models like Claude, GPT, Gemini or Mistral are non-deterministic, meaning the same input won’t always produce the same output. They don’t know how to read structured data easily.
- **[Use case ideas](https://docs.dust.tt/docs/use-cases):** Examples like writing, coding, research, translation, idea generation, and task automation.
- **[Agents](https://docs.dust.tt/docs/quickstart-agent):** What are Dust agents? How do I use or create one?
- **[Data Analysis:](https://docs.dust.tt/docs/table-queries)** How can I use Dust Table Query tool for data analysis?
