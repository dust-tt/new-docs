## What do users need to understand to get started?

- **Why Dust:** Why did my company decide to use Dust, and how am I expected to use it?
- **Capabilities:** What can Dust do?
- **Compliance Guidelines:** Are there specific documents or data we shouldn’t use with Dust?
- **Data Usage:** What data is available in Dust?
- [**Instructions:**](/docs/user-documentation/agents/llm-best-practices/how-to-write-effective-instructions) How do I create good instructions?
- [**Non-Deterministic Outputs and LLMs limitations**](/docs/user-documentation/agents/llm-best-practices/understanding-llm-limitations): Large language models like Claude, GPT, Gemini or Mistral are non-deterministic, meaning the same input won’t always produce the same output. They don’t know how to read structured data easily.
- **[Use case ideas](/docs/user-documentation/getting-started/use-cases-and-guides/index):** Examples like writing, coding, research, translation, idea generation, and task automation.
- **[Agents](/docs/user-documentation/agents/create-your-first-agent):** What are Dust agents? How do I use or create one?
- **[Data Analysis:](/docs/user-documentation/agents/knowledge/table-queries)** How can I use Dust Table Query tool for data analysis?
