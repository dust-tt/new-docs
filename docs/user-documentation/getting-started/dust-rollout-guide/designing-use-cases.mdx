Get guidance on designing valuable Dust use cases and building custom Dust agents and workflows for your organization.

### What's possible with Dust?


You can create custom agents to act as copilots for every team. Identifying teams needs is a good first step to ensure adoption and immediate impact.

## Sharing use case inspiration

Discover [Dust use cases](/docs/user-documentation/getting-started/use-cases-and-guides/index) and share them with your team

## Identifying and prioritizing high-value use cases

We recommend starting with 3 to 5 low effort, high value add use cases, ideally one for each team involved in the pilot or onboarding.

Focusing on low-effort, high-value tasks allows you to deploy Dust quickly and achieve immediate results. By starting with simple integrations and automations, you can demonstrate value while building momentum for more complex implementations.
