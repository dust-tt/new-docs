## Go further with 3rd party MCPs

Connect context from any MCP server to help your engineering agents work with more of your stack.

- [GitHub](https://dust.tt/integrations/github): Automates engineering workflows around issues, PRs, discussions, PR reviews, issue comments, and PR summaries or changelog entries.
- [Linear](https://dust.tt/integrations/linear) & [Jira](https://dust.tt/integrations/jira): Helps engineering teams create, update, search, and summarize issues/projects.
- [Statuspage](https://dust.tt/integrations/statuspage): Lets agents create and update customer-facing incident updates from internal incident context.
- [Supabase](https://dust.tt/integrations/supabase): Supports backend and database operations, useful for querying project state, debugging backend issues, and managing Supabase-powered apps.
- [Val Town](https://dust.tt/integrations/val_town): Lets agents run lightweight TypeScript/JavaScript automations, call APIs, transform data, and prototype small internal tools.
- [Amplitude](https://dust.tt/integrations/amplitude): Helps engineers investigate product behavior, event regressions, experiments, cohorts, session replays, and user-level product issues.
- [Snowflake](https://dust.tt/integrations/snowflake) / [BigQuery](https://dust.tt/integrations/bigquery) / [Databricks](https://dust.tt/integrations/databricks): Lets engineering teams ask natural-language questions over production or product data to debug incidents, validate migrations, or inspect user/workspace state.
- [Costory](https://www.costory.io/): Helps engineering and platform teams investigate cloud or AI spend, compare costs over time, create alerts, and generate FinOps reports.
